1. ATRIBUTOS DE UN PROYECTO

DEFINICIÓN FUNDAMENTAL

Un proyecto es un esfuerzo temporal emprendido para crear un producto, servicio o resultado único.

🎯 ATRIBUTOS PRIMARIOS

✅ Resultado Único

Produce productos, servicios o resultados que no han existido previamente en la organización.

Ejemplo: Implementar un sistema de autogestión de pagos en una alcaldía

⏱️ Naturaleza Temporal

Tiene fecha de inicio y fin definidas; concluye cuando se alcanzan los objetivos o se determina su inviabilidad.

Ejemplo: Proyecto web: inicio enero 2026, cierre agosto 2026

📋 ATRIBUTOS ADICIONALES

  • Progresividad: Se desarrolla por etapas y se refina continuamente
  • Recursos Limitados: Opera con restricciones de presupuesto, tiempo y personal
  • Riesgo e Incertidumbre: Involucra elementos no rutinarios que requieren gestión proactiva
Referencia: PMI. (2017). PMBOK® Guide (6ª ed.). ISBN: 978-1-62825-184-5

2. CICLO DE VIDA DE UN PROYECTO

FASES DEL CICLO DE VIDA

Serie de fases por las que pasa un proyecto desde su inicio hasta su cierre, proporcionando la estructura básica para la gestión.

1

🚩 Inicio

Definir el proyecto a alto nivel y obtener autorización

Ejemplo: Acta de Constitución para portal web
2

📋 Planificación

Establecer alcance, cronograma, presupuesto y riesgos

Ejemplo: Definir requerimientos de constancias laborales
3

⚙️ Ejecución

Realizar el trabajo definido en el plan

Ejemplo: Desarrollar plataforma web y capacitar personal
4

📊 Monitoreo y Control

Seguir, revisar y regular el progreso

Ejemplo: Verificar plazos, presupuesto y calidad
5

✅ Cierre

Finalizar todas las actividades y entregar resultados

Ejemplo: Entrega formal del sistema y documentación
Referencia: PMI. (2017). PMBOK® Guide, Sección 1.5 "The Project Life Cycle"

3. PROCESO DE GERENCIA DEL PROYECTO

DEFINICIÓN

La gerencia de proyectos es la aplicación de conocimientos, habilidades, herramientas y técnicas a las actividades del proyecto para satisfacer sus requisitos.

Iniciación

  • Desarrollar Acta de Constitución
  • Identificar Interesados

Planificación

  • Planificar alcance y cronograma
  • Planificar costos y calidad
  • Planificar recursos y riesgos

Ejecución

  • Dirigir y gestionar el trabajo
  • Gestionar calidad y equipo
  • Gestionar comunicaciones

Monitoreo y Control

  • Monitorear trabajo del proyecto
  • Control integrado de cambios
  • Validar y controlar alcance

Cierre

  • Cerrar proyecto o fase
  • Entregar resultados finales
  • Documentar lecciones aprendidas
Referencia: PMI. (2017). PMBOK® Guide, Sección 1.9 "Project Management Process Groups"

Habilidades y dinámicas humanas

  • Competencias técnicas
  • Liderazgo
